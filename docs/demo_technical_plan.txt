AI Classroom Assistant Demo Technical Plan

1. Project Goal

Build a lightweight classroom AI assistant demo for software competition presentation. The product is a web-based teacher and student system. Both the teacher side and the student side must be delivered as web pages.

The demo must keep the original project idea clear:

- It is an AI classroom interaction assistant for offline teaching
- It is similar in spirit to an AI version of classroom interaction tools, but the focus is not PPT control or random roll call
- Its core value is real-time classroom sensing, teaching strategy support, safe student Q and A, pressure sensing, and PPT-based mind map generation

The demo must be stable enough to build in about three days with Codex. It is a presentation-first demo, not a production system.


2. Original Project Idea Mapping

This demo must carefully reflect the creative ideas from the project draft.

2.1 Real-time classroom resonance heatmap
- Students send anonymous feedback in class
- Feedback is not shown as public bullet chat
- The system uses WebSocket real-time updates
- The teacher sees a heatmap, confusion rate, and trend view
- A dynamic threshold is used to judge understanding deficit
- When confusion rises, the teacher gets direct AI teaching suggestions instead of only raw numbers

2.2 AI answer verification with source note
- Student questions should not receive free and unsafe model output
- The answer must be grounded in local course material
- The answer area must show source notes under the answer
- If confidence is low, the answer style must become careful and conservative
- This is the demo version of the project idea about reducing model hallucination in teaching

2.3 Dynamic course pressure sensing
- The teacher can view pressure score, homework speed, accuracy, and mood words
- The UI should use weather style display such as sunny, cloudy, storm
- The system should show simple micro strategy suggestions for the next class
- A lightweight "knowledge simplification pack" idea should appear as a generated suggestion card in the pressure page

2.4 PPT to knowledge points and mind map
- The demo must include a feature that reads teaching PPT content
- The system summarizes key knowledge points from the PPT
- The system automatically generates a mind map
- The mind map must be visible on both the teacher side and the student side
- This feature is part of the demo scope and must not be removed

2.5 Classroom realism
- Student feedback should be anonymous
- A lightweight anti-fake rule should exist in the demo, such as cooldown time, duplicate limit, and simple session filtering
- The teacher should feel that AI is acting like a live teaching assistant

6. Product Flow

Flow A: Classroom feedback
1. Student opens the web feedback page
2. Student clicks one of four feedback buttons
3. Client sends feedback event through Socket.IO
4. Server updates in-memory session stats
5. Teacher dashboard web page updates charts and confusion rate in real time
6. If threshold is reached, teacher sees an alert and strategy cards

Flow B: Student ask with source note
1. Student opens the web ask page
2. Student enters a question in Chinese
3. API loads local course chunks from JSON
4. Retrieval picks the top matching chunks
5. API sends question and chunks to DeepSeek
6. API returns answer, source notes, and confidence level
7. UI shows answer plus source note card in Chinese
8. If confidence is low, UI shows safe fallback text in Chinese

Flow C: Pressure display
1. Teacher opens the web pressure page
2. Page loads mocked class data from JSON
3. System computes a simple pressure score
4. UI shows weather-style status and micro strategy tips in Chinese
5. UI shows one lightweight knowledge simplification pack card in Chinese

Flow D: PPT to mind map
1. Teacher opens the web mind map page
2. Teacher uploads a .pptx teaching file
3. API extracts slide text from the PPT
4. API summarizes key knowledge points
5. API generates mind map nodes and edges
6. Teacher side shows the generated mind map
7. Student side reads the latest generated mind map and shows the same learning structure

10. Core Logic Rules

10.1 Confusion rate
Use this simple formula:

confusionRate =
(too_fast_count + confused_count) / all_feedback_count

10.2 Threshold alert
Trigger alert when all conditions are true:
- total feedback in current window >= 8
- confusionRate >= 0.30
- confusionRate - lastWindowRate >= 0.10

Window rule
- current window: last 30 seconds
- last window: previous 30 seconds

10.3 Strategy generation
Use local strategy-library.json first.
Select current topic.
Pick three strategy items by type:
- analogy
- visual
- quick_check

Then call DeepSeek only to rewrite them into natural teacher-facing Chinese text.
Do not let the model invent new teaching facts without library support.

The strategy card style should clearly reflect the original project idea:
- one visual suggestion
- one classic analogy suggestion
- one fast anonymous check suggestion

10.4 Ask answer confidence
Use a simple three-level confidence result.

High
- at least 2 matched chunks
- top score is strong

Medium
- 1 matched chunk with acceptable score

Low
- no clear chunk match

Low confidence output rule
- answer must use careful wording in Chinese
- add safe note in Chinese
- example safe note: "根据当前课程资料，这一部分可能还需要结合老师课堂讲解进一步确认。"

10.5 Anti-fake feedback rule
This is a lightweight demo rule only.

Rules
- same student can send only one feedback in 3 seconds
- if the same feedback type repeats too many times in a short window, keep only the latest one
- sessionId and studentId must exist

10.6 PPT to mind map rule
- Support only .pptx upload in the demo
- Extract text from slide title and body text
- Ignore images, animation, and complex layout
- Build knowledge points from extracted slide text
- Use DeepSeek to rewrite summary points into clean Chinese phrases
- Generate a simple tree-style mind map
- Save the latest result to local JSON for student and teacher pages

13. Retrieval Design

Use a simple local retrieval method that works with Chinese content.

Step 1
- Read the student question as Chinese text

Step 2
- Score each chunk by:
  - topicId exact match
  - keyword overlap count
  - title phrase match
  - content phrase match

Step 3
- Sort by score

Step 4
- Keep top 3 chunks with positive score

Step 5
- Send only these chunks to DeepSeek

Prompt rule
- Ask the model to answer only from provided material
- Ask the model to avoid unsupported claims
- Ask the model to answer in simple Chinese teaching language


14. PPT Summary And Mind Map Design

14.1 Input rule
- Accept .pptx file only
- Demo assumes text-based teaching PPT
- Demo does not support scanned image slides

14.2 Extraction rule
- Read slide title text
- Read slide body text
- Merge text by slide order
- Keep slide number for traceability

14.3 Summary rule
- Build 4 to 8 key knowledge points
- Each point must use short Chinese wording
- Use local topic hints first if topicId exists
- Let DeepSeek clean and organize the points

14.4 Mind map rule
- Root node is the course name or current topic
- First layer is major knowledge points
- Second layer can contain 1 to 3 short sub points when available
- Layout should be stable and readable

14.5 Shared display rule
- Teacher page shows upload, extraction preview, summary, and mind map
- Student page shows the latest summary and mind map without upload control
