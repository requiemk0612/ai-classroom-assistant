Runtime Integration Update

1. Purpose

This document records the second implementation pass after the first project skeleton.
The goal of this pass is to make the main demo pages closer to direct local run and page-level integration.
Every change in this pass has been checked against the requirements in the docs directory, especially docs/demo_technical_plan.txt.

- The required API routes still exist:

3. Main Integration Improvements In This Pass

- Student feedback page now shows socket connection status and feedback rejection status
- Teacher dashboard now pulls latest metrics from /api/metrics on mount
- Teacher dashboard now requests strategy data from /api/strategy when alert conditions appear
- Teacher pressure page now requests latest data from /api/pressure on mount
- Student ask page now uses shared API client helpers and clearer status text
- Student mind map page now requests latest data from /api/mindmap on mount
- Teacher mind map page now requests latest data from /api/mindmap on mount
- Teacher mind map page now validates .pptx input more clearly before submit
- Retrieval output is now closer to a concise grounded answer instead of raw chunk dump
- PostCSS config was updated to match the ESM project setup
