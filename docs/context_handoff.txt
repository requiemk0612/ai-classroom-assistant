4. Exceptions And Abnormal Cases

Observed abnormal cases:
- Earlier, Node.js was installed but not available in PATH. This was repaired during the environment work.
- npm install initially failed because Next.js 15.0.0 did not accept stable React 19. The package was updated to a compatible Next.js 15.5.14 release.
- next build initially failed because some project files had UTF-8 BOM. Files were rewritten without BOM.
- A later npm run build command hung abnormally for a very long time. This is not normal for this project.
- The likely cause of the long hang is a stuck Node/Next child process, file locking, or Windows process issues rather than normal build time.
- Some commands were interrupted by the user during runtime verification, so there may have been temporary background processes at different moments.
- During one runtime test, API output viewed in PowerShell showed garbled Chinese text, but this was a terminal encoding display issue, not necessarily an app-side data corruption issue.
- Socket.IO basic metrics update was verified, but full browser-side visual interaction was not fully completed with real browser automation.

Important note about the long build hang:
- The project had already built successfully before.
- Therefore the later 30 minute build behavior should be treated as an abnormal stuck process, not expected build duration.
