Known limitation:
- Complex PPT files with heavy template styling, image-based text, or unusual layout metadata can still produce garbled extracted text.
- For the live demo, prefer a text-heavy .pptx with a simpler structure.


4. Exceptions And Abnormal Cases

Observed abnormal cases:
- Earlier, Node.js was installed but not available in PATH. This was repaired during the environment work.
- npm install initially failed because Next.js 15.0.0 did not accept stable React 19. The package was updated to a compatible Next.js 15.5.14 release.
- next build initially failed because some project files had UTF-8 BOM. Files were rewritten without BOM.
- A later npm run build command hung abnormally for a very long time. This was treated as a stuck process issue rather than expected build duration, and a clean .next reset resolved it.
- During one runtime test, API output viewed in PowerShell showed garbled Chinese text, but this was a terminal encoding display issue, not necessarily an app-side data corruption issue.
- During a later localhost probe, the running dev server briefly returned a development-only missing chunk error for mind map pages. This is most consistent with stale Next.js dev hot state and can be resolved by restarting the dev server.


5. Recommended Next Steps

Follow this order:
1. Restart the dev server before the final live presentation if the app has been edited for a long time.
2. Use a text-heavy .pptx sample for the mind map segment.
3. Keep the local fallback path available even when DeepSeek is enabled.
4. If future work continues, improve PPT parsing robustness for complex templates and image-heavy slides.
